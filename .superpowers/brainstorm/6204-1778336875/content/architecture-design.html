核心组件一：临床知识路由层（解决"推荐错药"）

路由结构：从口语到药品类别的四级跳转
层级 输入 输出 示例
L1 标准化 "拉肚子" 腹泻 (diarrhea) 口语→医学术语
L2 归类 腹泻 消化系统 · 感染性 疾病→系统+病因
L3 适应症映射 消化系统·感染性 抗感染药 · 肠道 系统→治疗类别
L4 药品召回 抗感染药·肠道 Rifaximin, Ciprofloxacin... 类别→具体药品
对比现有方案："腹泻"直接匹配到止泻药Loperamide，而非真正的抗感染药

核心组件二：患者口语增强映射（解决"推荐不出药"）

三层Fallback策略
1. 精确匹配：患者说"上呼吸道感染" → 直接命中标准术语
2. 同义词匹配：患者说"感冒" → 映射到"上呼吸道感染"
3. 症状→疾病推断：患者说"喉咙痛、流鼻涕、发烧" → 组合推理为上呼吸道感染
新增能力：当三层fallback都失败时，不返回空结果——而是根据症状关键词匹配最接近的药品类别，标记为"低置信度，建议医生审核"

安全过滤策略调整

原来 改为 理由
无适应症 → 硬排除 无适应症 → 标记"off-label"，审核后可用 医生审核把关
相对禁忌 → 软警告 相对禁忌 → 标记+降低排序优先级 保持可见但降低推荐力度
数据未验证 → 排除 数据未验证 → 标记+置于推荐列表末尾 医生可选择使用
绝对禁忌/过敏 → 硬排除 保持不变（硬排除） 安全底线不可退让

审核反馈闭环

系统推荐 → 医生审核 → ✅ 确认 ✏️ 修改 ❌ 拒绝
医生的每一次审核决策都记录为训练信号 → 优化知识路由的置信度权重 → 持续提升推荐质量
