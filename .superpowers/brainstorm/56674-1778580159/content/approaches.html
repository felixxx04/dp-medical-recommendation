Sankey 流向图优化方案

当选择多个疾病时，当前 240px 固定画布导致流向线和药物标签挤在一起。以下是三种优化方向：

A

动态高度 + 滚动容器

思路：画布高度根据选中疾病数量动态计算（每个节点约 30-40px），设最大高度 600px，超出则滚动。最简单直接。

优点

  • 改动最小，仅调高度逻辑
  • 保留完整三层流向
  • 直观，用户无需学习

缺点

  • 全选时仍需大量滚动
  • 药物多时右侧标签仍密集
  • 无法一眼看全貌

两层 Sankey + 右侧药物详情表

思路：Sankey只展示疾病→药物分类两层（去掉第三层药物节点），右侧用表格展示选中分类的具体药物和推荐次数。

优点

  • Sankey图始终清晰可读
  • 药物详情用表格更清晰
  • 全选疾病也没问题

缺点

  • 丢失了流向的完整三层
  • 需点击才能看药物详情
  • 布局改动较大
C

全屏弹窗 + 动态高度 Sankey

思路：卡片内保留缩略预览，点击"展开全屏"后在弹窗中展示完整三层Sankey，弹窗内画布高度自适应节点数量。

优点

  • 保留完整三层流向
  • 全屏空间充足
  • 卡片内不占过多空间

缺点

  • 多一步点击操作
  • 弹窗内药物标签多时仍需滚动
  • 需要实现弹窗组件
