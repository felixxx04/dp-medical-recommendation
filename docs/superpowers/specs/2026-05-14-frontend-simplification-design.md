# 前端页面信息精简设计

> 老师反馈：部分页面信息过多，看上去太杂乱。需要去掉/折叠/重组页面信息。

## 设计原则

1. **信息零丢失** — 所有被移走的信息都保留在详情面板/折叠区中，只从"默认展示"变为"按需查看"
2. **关键信息不隐藏** — 安全提醒、审核状态等影响使用的badge保留在主视图
3. **视觉层级分明** — 核心信息突出，技术细节收起，页面一眼能看到重点

---

## 1. DrugRecommendation 用药推荐（最杂乱）

### 当前问题
- 每张药卡堆叠7个inline badge（安全/证据/审核/模式/需审核/质量/DP警示）
- 评分分解行（原始评分→DP评分→噪声→置信区间条）挤在一行
- 推荐路径框占用额外空间
- DP对比表占大面积但非核心
- 排除药物列表默认全部展开
- 候选统计3卡+预算信息分开显示

### 精简措施

| 措施 | 类型 | 说明 |
|------|------|------|
| 药卡7badge → 4badge | 删除 | 只保留：安全等级、药物类别、需审核、DP噪声警示。其余3个（evidenceLevel/reviewStatus/mode/qualityWarning）移入详情面板 |
| DP对比表 → 详情面板 | 移入 | 点击药卡后在详情中查看，默认不展示 |
| 评分分解行 → 详情面板 | 移入 | 原始评分/DP评分/噪声delta/置信区间条全部移入详情面板 |
| 推荐路径框 → 详情面板 | 移入 | "推荐路径：疾病匹配→高血压→钙通道阻滞剂"移入详情面板 |
| 排除药物列表 → 默认折叠 | 折叠 | 改为"安全排除 3 种药物 ▾"，点击展开详情 |
| 候选统计3卡+预算 → 合并一行 | 合并 | "安全候选 3/1815 · 已排除 1812 · ε 0.05/10.0 (99.5%)" 一行文字 |
| 详情面板隐私说明 → 缩为一句 | 精简 | "本推荐由DeepFM模型生成，已施加差分隐私保护(ε=X)。结果仅供参考，请遵医嘱。" |

## 2. PatientRecords 患者管理

### 当前问题
- 统计4个独立卡片占用纵向空间
- 2个分布图默认展示（非日常核心）
- 卡片头部5+信息（BMI/过敏数/建档日期）+ 4个操作按钮
- 添加表单10字段一页展示
- 临床指标12字段无分组

### 精简措施

| 措施 | 类型 | 说明 |
|------|------|------|
| 统计4卡 → 合并一行 | 合并 | 单行横排数字展示：患者25 · 平均年龄58 · 慢病12 · 平均BMI 24.3 |
| 2个分布图 → 默认折叠 | 折叠 | "📊 年龄/疾病分布图 ▾"，点击展开 |
| 卡片头部精简 | 删除 | 去掉BMI、过敏数量、建档日期。BMI移入基本信息tab，过敏数量在过敏史tab中自然体现 |
| 操作按钮精简 | 删除 | 卡片头部只保留"推荐"按钮，编辑/删除移入展开后的tab区域 |
| 添加表单 → 分2步 | 重组 | ①基本信息（姓名/年龄/性别/身高/体重）→ ②健康档案（过敏/慢病/用药/病史） |
| 临床指标12字段 → 分3组 | 重组 | 器官功能（肾/肝）、心血管（血压/心率）+ 生活习惯（吸烟/饮酒）、代谢（血糖/HbA1c/胆固醇/LDL）。添加"已填3/6"进度提示 |

## 3. AdminDashboard 管理后台

### 当前问题
- 4大section纵向堆叠，页面很长
- 管理员身份卡 + 5个统计卡占空间
- 用户表格5列（含不常用的"最近登录"）
- 隐私账本3卡 + 进度条 + 过滤 + 表格 = 信息密集

### 精简措施

| 措施 | 类型 | 说明 |
|------|------|------|
| 4大区块 → Tab切换 | 重组 | 用户管理 / 模型训练 / 隐私账本，同一时间只显示一个区块 |
| 管理员卡 + 5统计卡 → 合并一行 | 合并 | "admin 管理员 · 患者25 · 用户5 · 今日推理8 · 推荐总数156 · ε剩余9.50" |
| ε摘要行 → 删除 | 删除 | 顶部统计行已有ε剩余，无需重复"已消耗ε:0.500 · 剩余ε:9.500" |
| 用户表格"最近登录"列 → hover提示 | 移入 | 鼠标悬停在用户名上时显示最近登录时间 |
| 账本3卡 → 合并到进度条旁 | 合并 | 进度条两端直接标注"已消耗0.50 / 总预算10.0"，省去3个独立小卡片 |

## 4. PrivacyConfig 隐私配置

### 当前问题
- 算法原理卡片默认展开，占用大量空间
- 4个滑块+各自说明框，纵向很长
- 实时指标4个独立卡片
- 研究内容8条对配置操作无帮助

### 精简措施

| 措施 | 类型 | 说明 |
|------|------|------|
| 算法原理卡片 → 默认折叠 | 折叠 | "📖 差分隐私算法原理 ▾"，点击展开 |
| 研究内容卡片 → 默认折叠 | 折叠 | "🔬 课题研究内容 ▾"，点击展开 |
| 4个滑块 → 核心2个 + 高级折叠 | 重组 | 默认展示ε和ε_total；δ/Δf/噪声机制/应用阶段折入"高级参数 ▾" |
| 实时指标4卡 → 合并一行+进度条 | 合并 | "噪声1.000 · 效用损失30% · 预算剩余9.50" + 一条进度条 |
| 滑块说明提示框 → 去掉 | 删除 | ε滑块下的"当前ε=1.00属于中等保护"蓝色框、ε_total下的说明框 → 改为tooltip提示 |

2. PatientRecords（表单体验差）
3. AdminDashboard（纵向太长）
4. PrivacyConfig（内容冗长但结构还行）
